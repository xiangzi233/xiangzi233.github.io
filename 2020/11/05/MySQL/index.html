1.什么是数据库

  • 1.概念

    • 数据库(DataBase，简称 DB)：将大量数据保存起来，通过计算机加工而成的可以进行高效访问的数据集
    • 数据库管理系统 (DataBase Management System，简称DBMS)：用来管理数据库的计算机系统
    • 数据库管理员（Database Administrator 简称DBA）：从事管理和维护数据库管理系统(DBMS)的相关工作人员

  • 2.数据库分类

    • 关系型：把复杂的数据结构归结为简单的二元关系（即二维表格形式），由各个有联系的表组成，比如MySQL
    • 非关系型（NoSQL）：比如Redis和MongoDB

  • 3.配置线上数据库

    • 在阿里云上购买RDS关系型数据库，ECS云服务器

      • 1.设置白名单为 0.0.0.0/0
      • 2.创建高权限账户
      • 3.创建数据库（注意字符集：utf8mb4）

2.CURD

  • 表

    • 表名：全英文小写；不用拼音；多个单词用_分隔；例：pay_record
    • 字段：每一列
      • 字段名：每一列的第一行（必须唯一）
      • 字段值：第二行及以下,读取数据从第二行开始，数组下标为0(可以为NULL)
    • 数据：表格中的每一行
    • 主键：
      • 是一个特殊字段
      • 表格可以没有主键，但最多只能有一个
      • 值不能为NULL
      • 值绝对唯一
      • 一般用主键和其他表关联

  • SQL常用数据类型

    类型 含义
    VARCHAR 可变的长字符串
    INT 整型
    DOUBLE 浮点型
    DATETIME 时间类型，长度为0，格式为YYYY-MM-DD HH:MM:SS，值为2019-12-31 23:59:59
    BIGINT 长整型

3.查询优化

  • LIKE 查询

  • AND & OR

    用在条件上，多条件查询

  • IN & NOT IN

  • NULL值

    概念：

    • NULL和NOT NULL是MySQL字段的数据类型的属性，不属于任何数据类型
    • NULL值不等于空值，占用存储空间

    处理NULL值：

    • IS NULL：当列的值是 NULL 时，返回 true
    • IS NOT NULL：当列的值不为 NULL，返回 true
    • <=>：比较操作符，当比较的两个值都为 NULL 时或者相等时，返回为 true

    注意：NULL=NULL会返回NULL，不可用 =NULL，或者!=NULL来查找NULL值

5.MySQL关联查询

  • 外连接：左连接LEFT JOIN（右连接RIGHT JOIN）

    侧重于一张表的数据，返回一张表的所有数据。

    • JOIN：关联查询关键词

    • LEFT：表示左连接，返回左表的所有数据，即使右表没有匹配的数据（此时右表会以 NULL 的形式匹配数据）

    • ON：关联查询条件，eacher.id = course.teacher_id 指teacher表的id字段和course的teacher_id字段相等并匹配

    • 表结构设计规范

      1. 必须有主键
      2. 一个字段只表示一个含义
      3. 总是包含两个日期字段:gmt_created(创建日期),gmt_modified(修改日期)，且这两个字段不应该包含有额外的业务逻辑
      4. MySQL 中，gmt_created、gmt_modified 使用 DATETIME类型。
      5. 禁止使用复杂数据类型（数组，自定义类型等）
      6. 禁止使用物理外键，使用逻辑外键。（逻辑外键：例如表 b 记录了表 a 的 id，我们只需在表 b 中添加一列：a_id 就可以了，然后通过程序来控制外键关系来关联两张表）
      7. 禁止物理删除，使用逻辑删除is_deleted （逻辑删除（英语：logical deletion），又被称软删除、假删除，是一种数据库操作，使用标记将数据标为不可用，而不从数据库删除数据本身。使用适当的方法可恢复被删除的数据。）

  • 多表关联查询

    选中一张表作为主表，以它为基准，进行LEFT或者RIGHT JOIN查询

    ABC多表查询的本质是A先和B关联查询，然后再和C进行关联查询

  • 内连接（INNER JOIN，　INNER可省略）

    两张表之间共有的数据查询

  • UNION（两张表都全部包含）

    UNION/UNION DISTINCT：合并 并去重

    UNION ALL：合并，不去重
